Flux d'authentification

Deux flux distincts — création de compte par l'admin, puis connexion quotidienne du travailleur

ADMIN Flux 1 — Création d'un nouveau compte travailleur
👨‍💼
Admin RH crée le profil
Nom, prénom, service, bureau, option A/B, rôle
→
📧
Resend envoie l'invitation
Email avec code OTP 6 chiffres, valable 24h
→
🔢
Travailleur saisit le code
Sur la page d'activation de l'app
→
🔒
Travailleur crée son mot de passe
Min 8 caractères, confirmé
→
✅
Compte actif
Accès au portail personnel

WORKER / ADMIN Flux 2 — Connexion quotidienne
🌐
Page de login
Email + mot de passe
→
🔐
Supabase Auth vérifie
Session JWT sécurisée
→
🏠
Dashboard personnalisé
Selon rôle : worker ou admin_rh
SYSTÈME Flux 3 — Mot de passe oublié
❓
Clic "Mot de passe oublié"
→
📧
Code OTP envoyé par email (Resend)
→
🔒
Nouveau mot de passe défini
Note technique : Supabase Auth gère nativement les OTP et les invitations. Resend est utilisé uniquement pour le template email personnalisé (logo CG FGTB, texte en français). Pas de SMS — email uniquement.
